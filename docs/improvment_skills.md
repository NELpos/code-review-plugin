# Skill 문서 개선 기준

## 목적

`skills/` 하위 Skill 문서를 일관된 품질 기준으로 유지하기 위한 체크리스트다.

## 핵심 원칙

1. `description`은 트리거 전용으로 작성한다.
2. `SKILL.md body`는 실행 절차와 산출물 규격만 작성한다.
3. 본문은 500줄 이하를 목표로 유지한다.
4. 상세 지식은 `references/`로 분리하고, `SKILL.md`에는 로딩 경로만 둔다.
5. 100줄 이상 참조 문서는 상단에 `목차`를 둔다.
6. 지시 문구는 모호한 표현보다 `MUST/SHOULD`를 우선 사용한다.
7. 복잡 작업은 `실행 → 검증 → 수정 → 재검증` 루프를 본문에 포함한다.
8. 대규모/다중영역 작업은 sub-agent 위임 조건과 통합 규칙을 본문에 명시한다.

## 금지 패턴

- SKILL 본문에 "이 Skill은 언제 사용" 같은 트리거 설명 반복
- YAML frontmatter에 `name`, `description` 외 필드 추가
- 본문과 참조 문서에 같은 설명 중복
- 장문 개념 설명 위주 구성 (실행 지침/예시 부족)

## 리뷰 체크리스트

- Frontmatter는 `name`, `description`만 포함하는가?
- Description이 트리거 문맥과 작업 범위를 충분히 포함하는가?
- Body가 실제 실행 순서 중심으로 작성되어 있는가?
- 출력 형식(리포트 구조, 우선순위, 근거 표기)이 명시되어 있는가?
- 참조 문서 분리와 경로 안내가 명확한가?
- 긴 문서의 탐색성(목차, 섹션 구조)이 확보되어 있는가?
- sub-agent 위임 기준(언제 위임/언제 직접 처리)이 명시되어 있는가?
- sub-agent 출력 계약(필수 필드, 근거 방식)이 정의되어 있는가?
